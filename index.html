Platform Manual

Local Configuration & Multi-Agent Telemetry Integration Pipeline.

01. Core Capabilities

CrowObservability provides centralized, sovereign log ingestion and compliance verification for local software pipelines, continuous integration nodes, and autonomous AI agents (LangGraph, CrewAI, Autogen). It intercepts security anomalies, monitors local Model Context Protocol (MCP) networks, and flags code defects before artifacts reach registries.

02. Local Vault Setup (.env)

To secure the data collection layer, configure a text file named .env inside your root app folder:

PORT=3000
CROW_API_KEY=your_secure_custom_token_here

03. Compliance & Policy Mappings

🛡️ SOC 2 Type II Auditing Flags plaintext PII transfers, unmasked AI prompt dumps, and missing data-encryption guardrails.
📦 Supply Chain Sec Tracks dependencies, untrusted external runtime tools, and non-verified third-party scripts.

04. Connecting AI Pipelines & Tools

Configure pipelines to send a standard JSON metadata array via an HTTP POST request authenticated by a Bearer Token:

Target Endpoint: POST http://localhost:3000/api/report-compliance

Required Header: Authorization: Bearer [your_secure_custom_token_here]
